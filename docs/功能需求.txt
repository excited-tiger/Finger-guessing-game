# 十五二十猜拳游戏模拟软件需求说明

## 1. 产品概述
这是一个智能猜拳模拟器，实现了传统的十五二十猜拳游戏的人机对战。系统集成了语音识别、实时手势识别和自动判定等功能。

## 2. 功能需求

### 2.1 基础功能
- **游戏模式**：人机对战
- **对战局数**：5局有效比赛（平局不计入）
- **操作方式**：手势识别 + 语音指令

### 2.2 手势规则
- **单手可选数值**：
  - 0（握拳）
  - 5（张开）
- **双手组合**：
  - 0 + 0
  - 0 + 5
  - 5 + 5

### 2.3 语音控制
- **支持指令**：
  - "开始游戏"
  - "退出游戏"
  - "15 15"（开始对战）

### 2.4 胜负判定机制
- **玩家胜利条件**：玩家喊数等于双方手势总和，且与机器人喊数不同
- **机器人胜利条件**：机器人喊数等于双方手势总和，且与玩家喊数不同
- **平局条件**：双方喊数相同或均未猜中手势总和

## 3. 界面设计

### 3.1 主界面布局
- **中央区域**：机器人动态手势显示（白色背景）
- **右上角**：实时状态提示区
- **右下角**：玩家手势捕捉窗口
- **功能按钮**：
  - 开始游戏
  - 退出游戏
  - 游戏规则

### 3.2 数据显示
- **实时统计**：
  - 玩家胜局数
  - 机器人胜局数
  - 平局次数
- **操作记录**：最近3次出拳记录（包含双方手势和喊数）

## 4. 游戏流程

### 4.1 开局流程
1. 玩家通过语音"15 15"或点击按钮开始
2. 系统展示3秒准备动画
3. 显示双方手势

### 4.2 对战流程
- 每局限时15秒
- 机器人随机生成手势（0/5各50%概率）
- 出拳前保持1秒静止

### 4.3 结束处理
- **正常结束**：完成5局后显示战报
- **提前退出**：显示当前统计数据
- **结束选项**：
  - 继续（重置数据）
  - 退出（返回主界面）

## 5. 技术指标

### 5.1 性能要求
- 手势渲染帧率：60fps
- 指令响应延迟：200ms
- 实时手势识别和渲染

### 5.2 异常处理
- 无效输入自动重新检测
- 超时情况下随机生成有效手势
